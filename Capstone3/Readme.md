## 🧱 Architecture — Component by Component

### Token Embedding + Positional Encoding

```
token_ids (B, T) ──→ nn.Embedding ──→ (B, T, d_model)
                                              +
                       Sinusoidal PE  ──→ (B, T, d_model)   [classic]
                    or Learned PE     ──→ (B, T, d_model)   [GPT-2]
                    or RoPE           ──→ applied inside attn [LLaMA]
```

### Scaled Dot-Product Attention

```
      Q (B, H, T, d_k)                    K (B, H, T, d_k)
           │                                     │
           └──────────────────────┐              │
                                  ▼              ▼
                          S = Q @ K^T / sqrt(d_k)     (B, H, T, T)
                                  │
                          apply causal mask (−∞ to future)
                                  │
                          softmax(S) = attention weights
                                  │
                          output = weights @ V           (B, H, T, d_v)
```

### Multi-Head Attention

```
x (B, T, d_model)
   │
   W_QKV (single fused linear, d_model → 3×d_model)
   │
   split → Q, K, V each (B, T, d_model)
   reshape → (B, n_heads, T, d_k)   d_k = d_model / n_heads
   │
   [RoPE rotation applied here if use_rope=True]
   │
   scaled_dot_product_attention ×n_heads  (parallel)
   │
   merge heads: (B, T, d_model)
   │
   W_O projection: (B, T, d_model)
```

### Decoder Block

```
x ─────────────────────────────────────────── (+) ──→ x'
│                                              ↑
└──→ LayerNorm → MultiHeadAttention ──────────┘

x' ────────────────────────────────────────── (+) ──→ output
│                                              ↑
└──→ LayerNorm → FeedForward ─────────────────┘
```

### Full GPT Model

```
tokens (B, T) ──→ TokenEmbed + PosEnc ──→ (B, T, d_model)
                                                │
                                     ┌──────────┤
                                     │ Block 0  │  ×N layers
                                     │ Block 1  │
                                     │   ...    │
                                     │ Block N  │
                                     └──────────┤
                                                │
                              Final LayerNorm ──┤
                                                │
                              LM Head Linear ───→ (B, T, vocab_size)
                              (weights tied to token embed)
```

## ⚡ Modern Variants: Classic vs LLaMA-style

| Component | Classic (GPT-2) | Modern (LLaMA) |
|-----------|----------------|----------------|
| Position  | Learned PE (additive) | RoPE (rotational, in Q/K) |
| Norm      | LayerNorm | RMSNorm (no mean subtraction) |
| FFN       | Linear → GELU → Linear | SwiGLU: `(xW₁ ⊙ SiLU(xW₃)) W₂` |
| Attention | torch SDPA | torch SDPA (FlashAttention) |
| Bias      | Yes (in all layers) | No bias anywhere |

## 🔑 Key Concepts You'll Master

### Attention
| Concept | The "Aha" Moment |
|---------|-----------------|
| Permutation invariance | Without PE, [A,B,C] == [C,A,B] — no concept of order |
| QKV projections | Q="what I need", K="what I offer", V="what I carry" |
| sqrt(d_k) scaling | Without it: large d_k → dot products explode → softmax saturates |
| Causal mask | Can't see the future during training — lower-triangular −inf mask |
| Multi-head | Each head learns a different relationship type in parallel |

### Positional Encoding
| Concept | The "Aha" Moment |
|---------|-----------------|
| Sinusoidal PE | Fixed frequencies, unique at every position, some length generalisation |
| Learned PE | Trainable, slightly better but fails beyond max_len |
| RoPE | Rotation applied to Q/K → `dot(Q_m, K_n) = f(q, k, m-n)` (relative distance only) |
| RoPE extrapolation | Frequencies are math, not lookup → can extend beyond training length |

### FFN / Activations
| Concept | The "Aha" Moment |
|---------|-----------------|
| FFN role | "Associative memory" — stores and retrieves facts via weights |
| GELU vs ReLU | GELU smooth, non-zero for small negatives, better gradients |
| SwiGLU | Gate path learns WHEN to pass info; value path learns WHAT to pass |

### FlashAttention
| Concept | The "Aha" Moment |
|---------|-----------------|
| HBM vs SRAM | HBM = 16 GB slow VRAM; SRAM = 16 MB fast on-chip |
| Naive O(T²) | Full attention matrix in HBM: 128 GB at T=32k (32 heads) |
| Flash O(T) | Tile K/V, compute in SRAM, never write T×T to HBM |
| Online softmax | Running max+sum — computes softmax without seeing all scores first |

### KV Cache
| Concept | The "Aha" Moment |
|---------|-----------------|
| Training (parallel) | All T tokens processed at once → teacher forcing |
| Inference (sequential) | One new token per step → need all past K,V |
| Without cache | O(n²) compute for n tokens generated |
| With cache | O(n) compute: append new K,V, attend over all past |

## 🏋️ Training Configuration

| Config | Style | Params | Context | Time on RTX 2000 |
|--------|-------|--------|---------|-----------------|
| `gpt_nano` | Classic (GELU + LayerNorm + Learned PE) | 10.8M | 256 | ~15 min |
| `gpt_modern` | Modern (SwiGLU + RMSNorm + RoPE) | varies | 256 | ~20 min |

Both train on Tiny Shakespeare (1.1M chars, ~1M tokens) using:
- AdamW (β₁=0.9, β₂=0.95)
- Cosine decay with linear warmup
- Gradient clipping at 1.0
- AMP (float16 on CUDA)

Expected validation loss after 5000 steps: **~1.45** (character-level cross-entropy)

## ✅ Learning Checklist

After Capstone 3, you should be able to explain:

**Attention:**
- [ ] Why transformers need positional encoding (permutation invariance)
- [ ] What Q, K, V represent and why they're different projections
- [ ] Why attention scores are divided by √d_k
- [ ] How causal masking prevents the model from "cheating"
- [ ] Why multiple heads are useful (different relationship types)

**Modern Variants:**
- [ ] Why RoPE enables better length generalisation than learned PE
- [ ] Why `dot(Q_m, K_n)` with RoPE only depends on `m-n`
- [ ] What RMSNorm removes from LayerNorm and why that's OK
- [ ] The three weight matrices in SwiGLU and what the gate does

**FlashAttention:**
- [ ] Why naive attention is memory-bound, not compute-bound
- [ ] What HBM vs SRAM means and why the distinction matters
- [ ] How online softmax enables O(T) memory attention
- [ ] When FlashAttention helps the most (large T)

**KV Cache:**
- [ ] Why naive autoregressive generation is O(n²)
- [ ] How caching K,V reduces it to O(n)
- [ ] What memory a KV cache uses (formula)
- [ ] Why MQA/GQA (grouped-query attention) reduces KV cache size
